Build Your First Skill
How to Create a Skill

A Skill is just a markdown file with a clear name and a detailed prompt. You don't need to code anything. Here's the exact process:

1
Identify a Task You Do Repeatedly
Think about what you type into Claude most often. Writing ad hooks? Drafting client check-in emails? Building weekly content calendars? If you've explained it more than twice, it should be a Skill.
2
Write the Skill Prompt in a .md File
Open a text editor and write your full instructions as if briefing a new assistant. Include: what to do, what format to output, your brand voice rules, what to never do, and an example output if you have one. The more specific the Skill, the better Claude performs.
3
Add a Trigger Description
At the top of your .md file, add a short description that tells Claude when to activate this Skill ГўВЂВ” specific phrases, keywords, or situations that should load it automatically. This is what makes Skills feel magical: Claude knows when to use them without you typing the slash command.
4
Save It to Your Skills Folder in Cowork
In Cowork mode, save the file to your .claude/skills/ folder. From that point on, every Cowork session can access it. No setup, no re-loading ГўВЂВ” it's always there.
5
Test It and Refine
Run your Skill a few times. If Claude is missing something, go back and add it to the prompt. Skills get better every time you edit them. Think of it like training a team member ГўВЂВ” the more context you give, the less you have to correct.

Key Distinction
Skills vs. Projects vs. Prompts

These are three different layers of Claude's memory and instruction system. They all matter, but they do different things:

Claude Projects
Your Brand Context Layer
Lives in Claude's chat interface. Stores your system prompt + uploaded brand docs. Applies to all conversations inside that Project. Best for: brand voice, offer details, audience info.
Claude Skills
Your Workflow Automation Layer
Lives in Cowork on your computer. Stores specific task instructions. Triggered by keywords or slash commands. Best for: repeatable tasks, multi-step processes, file outputs.
How They Work Together
Use Projects for who you are (brand, offer, audience). Use Skills for what you do repeatedly (content, audits, emails). When they're both set up, Claude knows your business AND your workflows ГўВЂВ” every session starts at full speed.
